# Reference File Template

All reference files in `references/` should include YAML frontmatter for progressive disclosure loading.

## Required Frontmatter

```yaml
---
type: [core|workflow|quality|evaluation|optional]
load_when: [discovery|creation|quality|evaluation|optimization|distribution]
priority: [mandatory|recommended|supplemental]
estimated_lines: <number>
---
```

## Field Definitions

### type
- **core**: Foundational concepts required for all skill development work
- **workflow**: Specific workflow patterns and methodologies
- **quality**: Quality gates, compliance, and verification standards
- **evaluation**: Testing, benchmarking, and evaluation systems
- **optional**: Supplementary or reference material

### load_when
- **discovery**: Phase 1 - User asks about skill development concepts
- **creation**: Phase 2 - Building and implementing skills
- **quality**: Phase 3a/3b - Specification compliance and code quality
- **evaluation**: Phase 3.5 - Evaluation and optimization
- **optimization**: Improving existing skills
- **distribution**: Publishing and sharing skills

### priority
- **mandatory**: Always load for the specified phase
- **recommended**: Load by default, can be skipped for quick workflows
- **supplemental**: Load only on explicit request or for detailed workflows

### estimated_lines
- Approximate line count for context budgeting
- Helps the system determine whether to load the full file or a summary

## Execution Rule

The skill-ship skill uses these tags for lazy loading:

```yaml
Phase 1 (Discovery):
  load: type=core + load_when=discovery + priority=mandatory
  optional: type=core + load_when=discovery + priority=recommended

Phase 2 (Creation):
  load: type=core + load_when=creation + priority=mandatory
  load: type=workflow + load_when=creation + priority=mandatory
  optional: type=workflow + load_when=creation + priority=recommended

Phase 3a (Spec Compliance):
  load: load_when=quality + priority=mandatory

Phase 3b (Code Quality):
  load: type=quality + load_when=quality + priority=mandatory
  optional: type=quality + load_when=quality + priority=recommended

Phase 3c (Integration Verification):
  load: type=evaluation + load_when=evaluation + priority=mandatory

Phase 3.5 (Evaluation):
  load: type=evaluation + load_when=evaluation + priority=mandatory
  optional: type=evaluation + load_when=evaluation + priority=recommended
```

## Example

```markdown
---
type: quality
load_when: quality
priority: mandatory
estimated_lines: 450
---

# Skill Quality Gates

...
```

This file would be loaded during Phase 3b (Code Quality) as mandatory content.
